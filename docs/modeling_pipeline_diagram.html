3 Target & Split

회귀 Target

next_log_cells

  • 다음 조사 시점 세포수
  • log10(cells + 1)

분류 Target

target_alert_next

  • next_log_cells >= log10(1000 + 1)
  • 다음 시점 관심 이상 여부

Validation Strategy

date-level chronological holdout

  • 과거 날짜: train
  • 미래 날짜: valid

운영 관점 결론

현재 holdout 기준에서는 비트리 스케일 workflow가 회귀와 분류 모두 우세하다. 특히 조류경보는 미탐을 줄이는 Recall이 중요하므로 Logistic Regression이 강한 후보가 된다. 단, 트리 XGBoost는 Precision이 높아 오경보를 줄이는 비교군으로 유지한다.
